3. 설치할 때, 클라우드 부분에서 체크해제
   설치한 후, http://localhost 뿐만 아니라 http://127.0.0.1/index.html로도 접속이 가능해야 한다.
 - 웹 브라우저에서 index.html을 요청하여 웹 서버는 htdocs 폴더에서 index.html을 찾고 전달한다.
 - index.html을 전달받은 웹 브라우저는 이 코드를 읽어서 웹 브라우저에 화면을 띄울 수 있다.


4. 웹 서버와 파일

1) 웹 브라우저가 요청하여 웹 서버에게 받은 파일 읽어서 화면 열여주기
http://localhost/1.html
 - http : 웹 브라우저와 웹 서버가 서로 통신할 때 사용하는 통신규약(Hyper Text[웹 페이지] Transfer[전송하다] Protocol[규약])

2) 웹 브라우저에서 직접 파일을 읽어서 화면 열어주기(웹 서버를 통해 찾지 않는다.)
file:///C:/Bitnami/wampstack-7.4.7-0/apache2/htdocs/1.html


5. 웹 브라우저와 웹 서버의 통신

1) 웹 브라우저가 웹 서버에 요청할 때, 웹 서버의 주소(IP 주소)가 필요하다.

2) 윈도우라면, 윈도우에서 호스트의 IP주소를 확인한다.
 - cmd -> ipconfig
 - 실행 -> ncpa.cpl(네트워크 연결) -> 확인하고자 하는 네트워크 속성 -> 자세히 -> IPv4 주소 확인

3) 호스트의 IP주소를 이용하여 index.html을 요청한다.
 - 127.0.0.1은 루프백이므로 자기자신을 의미한다.
 - 만약 IP주소가 123.456.7.89라면 
   웹 브라우저의 도메인 주소창에 123.456.7.89/index.html을 검색하여 요청할 수 있다.

4) 스마트폰으로 웹 서버에 접속하기 전
   스마트폰은 웹 서버의 wifi와 동일한 wifi를 쓰고 있어야 한다.(무선을 쓴다면)

5) 만약 동일한 wifi를 쓰고 있는데
   웹 서버에 접속이 안 된다면 웹 서버의 와이파이 속성에서 네트워크 프로필이 공용인지 확인해야 한다.
   만약 네트워크 프로필이 공용이면 개인으로 변경해야 스마트폰에서 win10 아파치 웹 서버에 접속할 수 있다.

 - 스마트폰 ip 확인하는 방법(안드로이드 버전에서)
	설정 -> Wi-Fi -> Wi-Fi 고급설정 -> 스크롤을 내리면 정보에서 IP주소를 확인할 수 있다.
